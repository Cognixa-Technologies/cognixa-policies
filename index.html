Security Policy for Developing

Section 1: Contribution

  • All contributions must be made as pull requests, which can only be accepted after security checks the file. No Exceptions.
  • Intentionally Malicious code will be blame linked and the author(s) placed on warning or immediately removed.
  • Please hold backups of your work securely under some form of encryption [1] provided by Security.
  • Ensure your Environment Variables are encrypted STRICTLY under AES-GCM, using the shared key for Environment Variables.
  • Contributed code must have full explanations of changes and a short reference name that must strictly be Initials-Short Date-Version (For example OA-5.11.2024-0.1.0)

Section 2: Writing

  • All code must be checked by security before publishing as mentioned in section 1 point 1.
  • Written code must be fully debugged by the writer.
  • Methods like exec() are NEVER to be used around user input.
  • Attempt to use Context Handlers, Decorators, Functions and Type Checkers where ever possible to remove room for error. [2]

Section 3: Test Hosting

  • Test hosting instances of Cognixa Core must be done when the main instance is available, or on your own bot instance.
  • Be wary of sending data over public networks (avoid where possible)
  • If utilising a Dual-Wrapper [3] for the bot, please avoid leaving ports exposed on your device when you are not testing (for example if you are calling to a function server [4] on the main port)
